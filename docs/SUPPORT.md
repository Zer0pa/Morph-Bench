# Support

## Routing

| Need | Route |
|---|---|
| Reproducible bug | GitHub bug report |
| Evidence or claim dispute | GitHub evidence dispute |
| Feature request | GitHub feature request |
| General question | GitHub question template |
| Security issue | Private route in `../SECURITY.md` |

## What To Include

- repo version, branch, or package snapshot
- exact file or artifact path involved
- expected behavior versus observed behavior
- whether the issue concerns local staged proof or cited source-repo authority

## Response Expectations

| Request Type | Expected Handling |
|---|---|
| Bug | best-effort triage once reproduction and scope are clear |
| Question | best-effort async response |
| Evidence dispute | compare the claim to the named artifact or downgrade the claim |
| Feature request | review against `ROADMAP.md` and `SOURCE_BOUNDARY.md` |

## Boundary Note

Support replies do not change repo truth by themselves. Public claims change
only when the evidence and docs surface are updated.
