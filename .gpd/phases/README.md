# Phase Folder Guide

Each phase folder should contain:

- `<PHASE>-CONTEXT.md`
- `<PHASE>-RESEARCH.md`
- `<PHASE>-VERIFICATION.md`
- one or more numbered `<PLAN>.md` files
- matching numbered `<SUMMARY>.md` files

## Naming Pattern

- phase folder: `00-workstream-bootstrap`
- phase context: `00-CONTEXT.md`
- phase research: `00-RESEARCH.md`
- phase verification: `00-VERIFICATION.md`
- first plan: `00-01-PLAN.md`
- first summary: `00-01-SUMMARY.md`

Keep plans machine-readable at the top. Keep summaries honest and end-state
oriented. Add a new phase only after the roadmap names it.
