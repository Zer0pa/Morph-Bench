  <action>
    Extend `src/gnosis_morph_bench/stability.py` with three new modes
    plus an aggregator. Each mode's return payload MUST include a
    top-level `"mode"` key naming the mode.

    1. `noise_injection(manifest, route_name, config, *,
       noise_sigma: float = 0.05, repeats: int = 6) -> dict`:
       - Uses `extract_route_dataset` to get `(item_ids, labels, matrix)`.
       - Computes baseline cluster labels from
         `cluster_labels(StandardScaler().fit_transform(matrix), config.n_clusters, seed=config.seed)`.
       - For each repeat:
         - Draws Gaussian noise shape `matrix.shape` with
           `rng = np.random.default_rng(config.seed + offset)`.
         - Clusters `baseline_scaled + noise_sigma * noise`.
         - Computes `pairwise_jaccard(baseline_by_item, noisy_by_item, item_ids)`.
       - Returns `{ "mode": "noise_injection", "route_name": ...,
         "noise_sigma": ..., "repeats": ..., "mean_jaccard": ...,
         "min_jaccard": ..., "max_jaccard": ... }` with each value
         rounded to 6 decimal places.

    2. `k_sensitivity(manifest, route_name, config, *,
       k_offsets: tuple[int, ...] = (-1, 1)) -> dict`:
       - Computes baseline labels at `config.n_clusters`.
       - For each offset, computes labels at `k = max(2, min(len(item_ids) - 1, config.n_clusters + offset))`.
         (Record `requested_k` and effective `k` separately so callers
         see when clipping happened.)
       - Computes `pairwise_jaccard(baseline_by_item, shifted_by_item, item_ids)`.
       - Returns `{ "mode": "k_sensitivity", "route_name": ...,
         "k_base": config.n_clusters, "offsets": [...], "entries": [
           { "requested_k": k_base + offset, "effective_k": effective,
             "jaccard": ... }, ... ] }`.

    3. `seed_variance(manifest, route_name, config, *,
       seeds: tuple[int, ...] = (1, 2, 3, 4)) -> dict`:
       - Computes baseline labels at `config.seed`.
       - For each seed in `seeds`, re-runs the baseline via
         `dataclasses.replace(config, seed=new_seed)` and
         `evaluate_route` (cheap reuse), and computes Jaccard versus
         baseline across the shared item ids.
       - Returns `{ "mode": "seed_variance", "route_name": ...,
         "baseline_seed": config.seed, "seeds": [...],
         "jaccards": [...], "mean_jaccard": ..., "min_jaccard": ...,
         "max_jaccard": ... }`.

    4. `stability_battery(manifest, route_name, config, *,
       noise_sigma: float = 0.05, noise_repeats: int = 6,
       k_offsets: tuple[int, ...] = (-1, 1),
       seeds: tuple[int, ...] = (1, 2, 3, 4),
       fraction: float = 0.25,
       fraction_repeats: int = 6,
       replay_repeats: int = 3) -> dict`:
       - Calls all five modes and returns a dict keyed by mode name:
         ```
         {
           "route_name": route_name,
           "modes": {
             "deterministic_replay": {...},
             "leave_fraction_out": {...},
             "noise_injection": {...},
             "k_sensitivity": {...},
             "seed_variance": {...},
           }
         }
         ```
       - Also tag existing `deterministic_replay` and `leave_fraction_out`
         payloads with `"mode"` at the top level (edit those two
         functions minimally to add the key).

    5. Preserve the NaN/contract behavior of `extract_route_dataset`.
       If any mode encounters a route with NaN entries, the underlying
       `extract_route_dataset` call raises; do not catch it.

    6. Do NOT import from `phase3_common`, `stroke_*`, any monorepo
       module. Do NOT use `sys.path.insert` or `Path(__file__).parents[N]`.
  </action>
