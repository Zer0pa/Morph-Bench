       - `DEFAULT_MANIFEST_NAME = "indus-phase4"`
       - `DEFAULT_GOVERNING_ROUTE = "pixel_full_concat_31d_single"`

    2. `build_parser() -> argparse.ArgumentParser` with the required
       and optional arguments declared in the contract:
       - `--feature-manifest` (path, required)
       - `--reference-frozen` (path, required)
       - `--output` (path, default `DEFAULT_OUTPUT`)
       - `--manifest-name` (string, default `DEFAULT_MANIFEST_NAME`)
       - `--reference-key` (string, default resolves from the frozen
         reference payload; adapter loads `reference_frozen` and uses
         its declared key if `--reference-key` is not passed)
       - `--include-route` (string, repeatable, `action="append"`)
       - `--drop-item-with-nan` (flag)
       - `--run-record` (path; default derived from `--output` by
         replacing the extension with `_run.json`)

    3. Core translation function
       `translate(feature_manifest: dict, reference_frozen: dict, *,
       reference_key: str, include_routes: list[str] | None,
       drop_item_with_nan: bool, manifest_name: str) -> tuple[dict, dict]`:
       - Returns `(benchmark_manifest_payload, run_record_payload)`.
       - `benchmark_manifest_payload` conforms to the JSON shape in
         the contract's "Output Shape" section (schema_version=1,
         manifest_name, items with item_id/reference_labels/route_features).
       - Every item carries exactly one key in `reference_labels`:
         the governing reference key.
       - Routes are included iff all items expose the route, all
         vectors are numeric, and vector lengths are consistent. Routes
         that fail these checks are dropped with a `routes_dropped`
         entry in the run record.
       - If `--include-route` is passed at least once, only those
         routes pass the gate.
       - NaN/`null` in a route vector raises a `ValueError` by default;
         with `drop_item_with_nan=True` the affected item is dropped and
         the drop is recorded in `items_dropped`.
       - Missing reference-label entries on an item are a hard error.
       - The function must never re-binarize features, never fetch
         remote data, and never reach outside the declared inputs.

    4. `compute_freeze(benchmark_manifest_payload: dict,
       reference_key: str) -> dict`:
       - Calls `load_manifest` on a tempfile write of the payload and
         then `freeze_reference`. This keeps `schema.py` as the single
         implementation of the freeze algorithm.

    5. `verify_upstream_freeze(neutral_freeze: dict, upstream: dict,
       reference_key: str) -> None`:
       - Reads upstream's declared `sha256` for the governing
         reference key. If it disagrees with `neutral_freeze["sha256"]`,
         raises `ContractViolation` with a message naming the
         reference key and both hashes. Otherwise returns `None`.

    6. `run(args: argparse.Namespace) -> int`:
       - Loads inputs with `_utils.read_json`.
       - Calls `translate(...)`, `compute_freeze(...)`,
         `verify_upstream_freeze(...)`.
       - Assembles the run record (schema in contract's "Adapter Run
         Record" section): `adapter`, `adapter_version`, `manifest_name`,
         `input_paths` (absolute), `input_hashes` via `sha256_file`,
         `items_in`, `items_out`, `items_dropped`, `routes_in`,
         `routes_out`, `routes_dropped`, `reference_freeze`, and
         `schema_version=1`.
       - Writes both files with `_utils.write_json`.
       - Returns `0` on success, `2` on contract violation (missing
         required input, NaN in strict mode, frozen-SHA mismatch),
         `1` on any unhandled exception. The run record must be
         written on both success and contract-violation paths.

    7. `main(argv: list[str] | None = None) -> int`:
       - Wires `build_parser().parse_args(argv)` into `run(args)` and
         catches `ContractViolation` to return exit code 2, and
         unexpected `Exception` to return exit code 1 with the
         traceback printed to stderr (no silent swallow).

    8. Do NOT import from `phase3_common`, `stroke_*`, or any monorepo
       module. Do NOT call `sys.path.insert(...)`. Do NOT derive paths
       from `Path(__file__).resolve().parents[N]`. Do NOT hardcode

       helpers come from `gnosis_morph_bench._utils`.

    9. Update `pyproject.toml`:
       - Add a new console script:
         `gnosis-morph-bench-adapter-indus-phase4 = "gnosis_morph_bench.adapters.indus_phase4:main"`
       - Do not change the existing `gnosis-morph-bench-smoke`
         entrypoint.
